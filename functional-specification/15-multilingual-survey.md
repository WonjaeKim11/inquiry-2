## 2. 개요

### 2.1 목적

본 문서는 Inquiry 다국어 설문(Multi-Language Surveys) 기능의 상세 기능 명세를 정의한다. 하나의 설문을 여러 언어로 번역하여 글로벌 대상으로 배포할 수 있도록 하는 Enterprise 전용 기능이며, 프로젝트 레벨의 언어 관리, 설문 레벨의 언어 활성화, RTL 지원, 언어 전환 UI 등을 포함한다.

### 2.2 범위

**포함 범위 (In-scope)**:
- 다국어 문자열 타입(TI18nString) 정의 및 유효성 검증
- 프로젝트 레벨 언어 CRUD 관리
- 설문 레벨 언어 활성화/비활성화
- 기본 언어(default) 및 보조 언어(secondary) 관리
- `?lang=` 쿼리 파라미터를 통한 Link Survey 언어 지정
- RTL(Right-to-Left) 언어 자동 감지 및 적용
- 언어 전환 UI(showLanguageSwitch)
- Enterprise 라이선스 검증 (multiLanguageSurveys 기능 플래그)
- 번역 필드 자동 관리 및 언어별 텍스트 검증

**제외 범위 (Out-of-scope)**:
- AI 기반 자동 번역 기능
- 번역 워크플로우 및 번역자 역할 관리
- 관리자 UI 자체의 언어 설정 (별도 i18n 시스템, FSD-030 참조)

### 2.3 대상 사용자

| 역할 | 관심 사항 |
|------|----------|
| 글로벌 시장 조사자 | 다국어 설문을 통해 글로벌 대상을 조사하고 언어별 응답 수집 |
| 설문 관리자 | 언어별 번역 관리, 기본 언어 설정, 설문별 언어 활성화 제어 |
| 응답자 | 자국어로 설문에 응답하고, 필요 시 언어를 전환 |
| 시스템 관리자 | Enterprise 라이선스 관리 및 프로젝트별 언어 리소스 설정 |

### 2.4 용어 정의

| 용어 | 정의 |
|------|------|
| TI18nString | 다국어 문자열 타입. `Record<string, string>` 형태의 키-값 쌍으로, 반드시 `"default"` 키를 포함해야 한다. |
| default 키 | TI18nString에서 기본 언어 값을 저장하는 고정 문자열 키. ISO 언어 코드가 아닌 `"default"` 리터럴을 사용한다. |
| 기본 언어 (Default Language) | 설문에서 `default=true`로 설정된 언어. 설문당 정확히 1개만 존재해야 한다. |
| 보조 언어 (Secondary Language) | 설문에서 `default=false`인 활성화된 언어. 0개 이상 존재할 수 있다. |
| RTL (Right-to-Left) | 오른쪽에서 왼쪽으로 읽는 언어 방향. 아랍어, 히브리어, 페르시아어, 우르두어 등이 해당한다. |
| Language Switch | 응답자가 설문 진행 중 언어를 전환할 수 있는 UI 컴포넌트. `showLanguageSwitch` 속성으로 제어한다. |
| Link Survey | 독립 URL로 접근 가능한 설문 유형. `?lang=` 쿼리 파라미터로 언어를 지정할 수 있다. |
| CUID2 | Collision-resistant Unique Identifier v2. 언어 및 프로젝트 식별자에 사용되는 ID 형식. |
| Enterprise 라이선스 | 다국어 설문 기능을 사용하기 위해 필요한 상용 라이선스. `multiLanguageSurveys` 기능 플래그로 활성화 여부를 판별한다. |
| Multi-Language Card | 설문 에디터에서 다국어 설정을 관리하는 UI 카드 컴포넌트. |

## 4. 상세 기능 명세

### 4.1 다국어 문자열 타입 (TI18nString)

#### 4.1.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-015-01 |
| 기능명 | 다국어 문자열 타입 (TI18nString) |
| 관련 요구사항 ID | FR-023 / 4.1 |
| 우선순위 | 필수 |
| 기능 설명 | 설문의 모든 텍스트 필드에 적용되는 다국어 문자열 타입을 정의한다. `Record<string, string>` 형태로, `"default"` 키가 기본 언어 값을 저장하며, 추가 언어는 ISO 언어 코드를 키로 사용한다. |

#### 4.1.2 선행 조건

- 없음 (기본 데이터 타입 정의)

#### 4.1.3 후행 조건

- 설문의 모든 텍스트 필드가 TI18nString 타입으로 저장된다.
- `"default"` 키에 기본 언어 텍스트가 반드시 존재한다.

#### 4.1.4 기본 흐름

1. 시스템은 설문의 텍스트 필드를 TI18nString 타입(`Record<string, string>`)으로 저장한다.
2. TI18nString 객체 생성 시 `"default"` 키에 기본 언어 텍스트를 필수로 포함한다.
3. 추가 언어가 있을 경우 ISO 언어 코드(예: `"de"`, `"ko"`, `"ar"`)를 키로 하여 번역 텍스트를 저장한다.

**데이터 구조 예시**:
```json
{
  "default": "What is your name?",
  "de": "Wie heissen Sie?",
  "ko": "이름이 무엇인가요?",
  "ar": "ما اسمك؟"
}
```

#### 4.1.5 대안 흐름

- **AF-01**: 단일 언어 설문인 경우, TI18nString에 `"default"` 키만 존재한다.

#### 4.1.6 예외 흐름

- **EF-01**: `"default"` 키가 누락된 경우, 유효성 검증 실패. 에러 메시지: `"I18n string must have a 'default' key"`.
- **EF-02**: `"default"` 키의 값이 빈 문자열인 경우, 유효성 검증 실패.

#### 4.1.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 동작 |
|---------|------|------|------|
| BR-01-01 | default 키 필수 | TI18nString 유효성 검증 시 | `"default"` 키가 없으면 검증 실패 반환 |
| BR-01-02 | default 키 고정 문자열 | TI18nString 키 정의 시 | `"default"`는 ISO 언어 코드가 아닌 리터럴 문자열로 고정 |
| BR-01-03 | 언어 키는 ISO 코드 | 보조 언어 키 설정 시 | `"default"` 이외의 키는 ISO 언어 코드 형식(예: `en`, `ko`, `ar-SA`)을 사용 |

#### 4.1.8 데이터 요구사항

**입력 데이터**:

| 필드명 | 타입 | 필수 여부 | 유효성 검증 규칙 |
|--------|------|----------|----------------|
| default | string | 필수 | 비어있지 않은 문자열 |
| [언어코드] | string | 선택 | ISO 언어 코드 형식의 키, 문자열 값 |

**출력 데이터**:

| 필드명 | 타입 | 설명 |
|--------|------|------|
| TI18nString | Record<string, string> | 언어 코드를 키로 하는 다국어 문자열 객체 |

#### 4.1.9 화면/UI 요구사항

- 설문 에디터에서 언어별 텍스트 입력 필드를 제공한다.
- 활성화된 언어에 대해 탭 또는 드롭다운으로 언어를 전환하여 번역을 입력한다.

#### 4.1.10 비기능 요구사항

- NFR-015-01: 다국어 문자열에 `"default"` 키 필수 (유효성 검증)

### 4.2 프로젝트 레벨 언어 관리

#### 4.2.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-015-02 |
| 기능명 | 프로젝트 레벨 언어 관리 |
| 관련 요구사항 ID | FR-023 / 4.2 |
| 우선순위 | 필수 |
| 기능 설명 | 프로젝트 설정의 Languages 페이지에서 해당 프로젝트에서 사용 가능한 언어를 CRUD(생성, 조회, 수정, 삭제)로 관리한다. 등록된 언어는 프로젝트 하위 모든 설문에서 공유된다. |

#### 4.2.2 선행 조건

- Enterprise 라이선스의 `multiLanguageSurveys` 기능 플래그가 활성화되어 있어야 한다.
- 사용자가 프로젝트 설정에 접근할 수 있는 권한을 보유해야 한다.

#### 4.2.3 후행 조건

- 언어가 프로젝트 레벨에 등록/수정/삭제된다.
- 등록된 언어는 해당 프로젝트의 모든 설문에서 선택 가능하다.

#### 4.2.4 기본 흐름

**언어 생성**:
1. 사용자가 프로젝트 설정 > Languages 페이지로 이동한다.
2. 사용자가 "언어 추가" 버튼을 클릭한다.
3. 시스템이 언어 코드(ISO) 입력 필드와 별칭(alias) 입력 필드를 표시한다.
4. 사용자가 언어 코드(예: `ko`)와 선택적으로 별칭(예: "한국어")을 입력한다.
5. 시스템이 CUID2 형식의 고유 ID를 자동 생성한다.
6. 시스템이 Language 엔티티를 생성하여 저장한다.

**언어 조회**:
1. 사용자가 프로젝트 설정 > Languages 페이지로 이동한다.
2. 시스템이 해당 프로젝트에 등록된 모든 언어 목록을 표시한다.

**언어 수정**:
1. 사용자가 언어 목록에서 수정할 언어를 선택한다.
2. 사용자가 언어 코드 또는 별칭을 수정한다.
3. 시스템이 변경 사항을 저장한다.

**언어 삭제**:
1. 사용자가 언어 목록에서 삭제할 언어를 선택한다.
2. 시스템이 삭제 확인 요청을 표시한다.
3. 사용자가 확인하면 시스템이 해당 언어를 삭제한다.

#### 4.2.5 대안 흐름

- **AF-02-01**: 사용자가 이미 등록된 언어 코드를 다시 추가하려는 경우, 시스템이 중복 안내 메시지를 표시한다.

#### 4.2.6 예외 흐름

- **EF-02-01**: Enterprise 라이선스가 없는 경우, Languages 페이지에서 UpgradePrompt를 표시한다.
- **EF-02-02**: 유효하지 않은 언어 코드 입력 시, 입력 필드에 오류 표시와 함께 유효한 코드 안내.

#### 4.2.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 동작 |
|---------|------|------|------|
| BR-02-01 | 프로젝트 단위 관리 | 언어 CRUD 수행 시 | 언어는 프로젝트 레벨에서 관리되며, 해당 프로젝트의 모든 설문에서 공유 |
| BR-02-02 | 언어 ID 형식 | 언어 생성 시 | CUID2 형식으로 자동 생성 |
| BR-02-03 | 언어 코드 형식 | 언어 생성/수정 시 | ISO 언어 코드 형식 준수 (예: `en`, `ko`, `ar-SA`) |

#### 4.2.8 데이터 요구사항

**Language 엔티티**:

| 필드명 | 타입 | 필수 여부 | 유효성 검증 규칙 |
|--------|------|----------|----------------|
| id | string (CUID2) | 필수 (자동 생성) | CUID2 형식 |
| code | string | 필수 | ISO 언어 코드 형식 (예: `en`, `ko`, `ar-SA`) |
| alias | string \| null | 선택 | 사용자 정의 별칭, null 허용 |
| projectId | string (CUID2) | 필수 | 유효한 프로젝트 ID 참조 |

#### 4.2.9 화면/UI 요구사항

- 프로젝트 설정 > Languages 페이지에 언어 목록 테이블 표시
- 각 언어 행에 코드, 별칭, 수정/삭제 액션 버튼 표시
- "언어 추가" 버튼으로 새 언어 등록 가능

#### 4.2.10 비기능 요구사항

- NFR-015-05: Enterprise 라이선스 필수

### 4.3 설문 레벨 언어 활성화

#### 4.3.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-015-03 |
| 기능명 | 설문 레벨 언어 활성화 |
| 관련 요구사항 ID | FR-023 / 4.3 |
| 우선순위 | 필수 |
| 기능 설명 | 개별 설문에서 프로젝트에 등록된 언어 중 어떤 언어를 활성화할지 관리한다. 설문에는 언어 배열로 관리되며, 정확히 하나의 기본 언어와 0개 이상의 보조 언어를 설정한다. |

#### 4.3.2 선행 조건

- 프로젝트에 1개 이상의 언어가 등록되어 있어야 한다.
- Enterprise 라이선스의 `multiLanguageSurveys` 기능 플래그가 활성화되어 있어야 한다.
- 다국어 활성화를 위해서는 프로젝트에 2개 이상의 언어가 등록되어 있어야 한다.

#### 4.3.3 후행 조건

- 설문에 언어 배열이 설정된다.
- 정확히 하나의 언어가 `default=true`로 설정된다.
- `enabled=true`인 언어만 응답자에게 노출된다.

#### 4.3.4 기본 흐름

1. 사용자가 설문 에디터에서 Multi-Language Card를 연다.
2. 시스템이 프로젝트에 등록된 언어 목록을 조회하여 표시한다.
3. 사용자가 기본 언어(Default Language)를 드롭다운에서 선택한다.
4. 시스템이 선택된 언어를 `default=true`, `enabled=true`로 설정한다.
5. 사용자가 필요한 보조 언어를 선택/토글하여 활성화한다.
6. 시스템이 선택된 보조 언어를 `default=false`, `enabled=true`로 설정한다.
7. 설문에 언어 배열이 저장된다.

#### 4.3.5 대안 흐름

- **AF-03-01**: 사용자가 다국어를 활성화하지 않는 경우, 설문은 단일 언어(default만)로 동작한다.

#### 4.3.6 예외 흐름

- **EF-03-01**: `default=true`인 언어가 0개이거나 2개 이상인 경우, 시스템이 유효성 검증 오류를 반환한다.
- **EF-03-02**: 프로젝트에 언어가 1개만 등록된 경우, "2개 이상 언어 필요" 안내 메시지를 표시하고 다국어 활성화를 차단한다.

#### 4.3.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 동작 |
|---------|------|------|------|
| BR-03-01 | 기본 언어 유일성 | 설문 언어 설정 시 | 정확히 1개의 언어만 `default=true`여야 함 |
| BR-03-02 | 활성 언어만 노출 | 설문 렌더링 시 | `enabled=true`인 언어만 응답자에게 제공 |
| BR-03-03 | 최소 2개 언어 필요 | 다국어 활성화 시 | 프로젝트에 2개 이상 언어가 등록되어야 다국어 활성화 가능 |

#### 4.3.8 데이터 요구사항

**SurveyLanguage 엔티티**:

| 필드명 | 타입 | 필수 여부 | 유효성 검증 규칙 |
|--------|------|----------|----------------|
| language | Language (참조) | 필수 | 프로젝트에 등록된 Language 엔티티 참조 |
| default | boolean | 필수 | 설문 내에서 정확히 1개만 `true` |
| enabled | boolean | 필수 | `true`: 응답자에게 노출, `false`: 비노출 |

**설문 언어 배열**: `SurveyLanguage[]`

#### 4.3.9 화면/UI 요구사항

- 설문 에디터 내 Multi-Language Card에서 설정
- Default Language 드롭다운, Secondary Language 토글 리스트 표시

#### 4.3.10 비기능 요구사항

- NFR-015-04: 언어 전환 시 기존 데이터 보존 (`enabled=false`로 비활성화, 삭제 아님)

#### 4.5.1 기능 개요

| 항목 | 내용 |

| 기능명 | 기본 언어 변경 |

#### 4.5.2 선행 조건

- 설문에 다국어가 활성화되어 있어야 한다.
- 변경 대상 언어가 프로젝트에 등록되어 있어야 한다.

#### 4.5.3 후행 조건

- 새 언어가 `default=true`로 설정된다.
- 기존 기본 언어는 `default=false`로 변경된다.
- 기존 번역 데이터는 보존된다.

1. 사용자가 Multi-Language Card의 Default Language 드롭다운에서 새 언어를 선택한다.
2. 시스템이 기존 기본 언어의 `default` 값을 `false`로 변경한다.
3. 시스템이 새로 선택된 언어의 `default` 값을 `true`로 설정한다.
4. 시스템이 변경 사항을 저장한다.

#### 4.5.5 대안 흐름

- **AF-05-01**: 새로 선택된 언어가 설문의 언어 배열에 없는 경우:

  2. 기존 기본 언어의 `default`를 `false`로 변경한다.

#### 4.5.6 예외 흐름

- **EF-05-01**: 프로젝트에 등록되지 않은 언어를 선택하려는 경우, 드롭다운에 해당 언어가 표시되지 않으므로 선택 불가.

#### 4.5.7 비즈니스 규칙

| BR-05-02 | 자동 추가 | 미등록 언어를 기본으로 선택 시 | 설문 언어 배열에 자동 추가 |
| BR-05-03 | default 유일성 유지 | 기본 언어 변경 시 | 항상 정확히 1개의 언어만 `default=true` |

### 4.6 보조 언어 추가/제거

#### 4.6.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-015-06 |
| 기능명 | 보조 언어 추가/제거 |
| 관련 요구사항 ID | FR-023 / 4.6 |
| 우선순위 | 필수 |
| 기능 설명 | 설문에 보조 언어(Secondary Language)를 추가하거나 제거(비활성화)한다. 언어 추가 시 설문의 모든 텍스트 필드에 해당 언어 키가 자동으로 추가되며, 비활성화 시 번역 데이터는 보존된다. |

#### 4.6.2 선행 조건

- 설문에 다국어가 활성화되어 있어야 한다.
- 추가하려는 언어가 프로젝트에 등록되어 있어야 한다.

#### 4.6.3 후행 조건

- **추가 시**: 해당 언어가 `enabled=true`, `default=false`로 설문에 추가되고, 모든 TI18nString 필드에 해당 언어 키가 빈 문자열로 초기화된다.
- **비활성화 시**: 해당 언어의 `enabled`가 `false`로 변경되고, 번역 데이터는 삭제되지 않고 보존된다.

#### 4.6.4 기본 흐름

**보조 언어 추가 (새 언어)**:
1. 사용자가 Multi-Language Card에서 보조 언어 목록 중 미추가 언어를 선택한다.
2. 시스템이 해당 언어를 설문 언어 배열에 추가한다 (`enabled=true`, `default=false`).
3. 시스템이 설문 내 모든 TI18nString 필드에 해당 언어 코드를 키로 하는 항목을 추가하고, 값을 빈 문자열(`""`)로 초기화한다.

**보조 언어 활성화 (기존 비활성 언어)**:
1. 사용자가 비활성화된 언어의 토글을 켠다.
2. 시스템이 해당 언어의 `enabled`를 `true`로 변경한다.
3. 기존 번역 데이터가 보존된 상태로 다시 활성화된다.

**보조 언어 비활성화**:
1. 사용자가 활성화된 보조 언어의 토글을 끈다.
2. 시스템이 해당 언어의 `enabled`를 `false`로 변경한다.
3. 번역 데이터는 삭제하지 않고 보존한다.

#### 4.6.5 대안 흐름

- 없음

#### 4.6.6 예외 흐름

- **EF-06-01**: 기본 언어(`default=true`)를 비활성화하려는 경우, 시스템이 비활성화를 차단한다 (기본 언어는 항상 활성 상태여야 한다).

#### 4.6.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 동작 |
|---------|------|------|------|
| BR-06-01 | 비활성화 시 보존 | 보조 언어 비활성화 시 | `enabled=false`로 변경, 번역 데이터는 삭제하지 않음 |
| BR-06-02 | 자동 키 추가 | 새 언어 추가 시 | 설문의 모든 TI18nString 필드에 해당 언어 키를 빈 문자열로 추가 |
| BR-06-03 | 기본 언어 비활성화 금지 | 기본 언어 토글 시 | `default=true`인 언어는 비활성화 불가 |

#### 4.6.8 데이터 요구사항

**입력 데이터**:

| 필드명 | 타입 | 필수 여부 | 설명 |
|--------|------|----------|------|
| languageId | string (CUID2) | 필수 | 추가/토글할 언어 ID |
| enabled | boolean | 필수 | 활성화 여부 |

#### 4.6.9 화면/UI 요구사항

- Secondary Language 목록에 각 언어별 토글 스위치 표시
- 토글 ON: 해당 언어 활성화 (응답자에게 노출)
- 토글 OFF: 해당 언어 비활성화 (번역 보존)

#### 4.6.10 비기능 요구사항

### 4.7 다국어 비활성화

#### 4.7.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-015-07 |
| 기능명 | 다국어 비활성화 |
| 관련 요구사항 ID | FR-023 / 4.7 |
| 우선순위 | 필수 |
| 기능 설명 | 설문의 다국어 기능을 완전히 비활성화한다. 모든 번역 데이터가 삭제되는 파괴적 작업이므로 확인 모달을 통한 사용자 동의가 필요하다. |

#### 4.7.2 선행 조건

- 설문에 다국어가 활성화되어 있어야 한다.

#### 4.7.3 후행 조건

- 설문의 languages 배열이 초기화된다.
- 모든 TI18nString 필드에서 `"default"` 키 이외의 번역 데이터가 삭제된다.

#### 4.7.4 기본 흐름

1. 사용자가 Multi-Language Card에서 다국어 비활성화를 요청한다.
2. 시스템이 확인 모달을 표시한다.
   - 제목: "Remove translations" (또는 동등한 경고 메시지)
   - 내용: 모든 번역 데이터가 삭제된다는 경고
   - 스타일: destructive (파괴적 작업 스타일)
3. 사용자가 확인 버튼을 클릭한다.
4. 시스템이 설문의 languages 배열을 초기화한다.
5. 시스템이 모든 TI18nString 필드에서 `"default"` 키 이외의 언어 키를 제거한다.

#### 4.7.5 대안 흐름

#### 4.7.6 예외 흐름

### 4.8 Language Switch UI

#### 4.8.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-015-08 |
| 기능명 | Language Switch UI |
| 관련 요구사항 ID | FR-023 / 4.8 |
| 우선순위 | 필수 |
| 기능 설명 | 응답자가 설문 진행 중 언어를 전환할 수 있는 UI를 제공한다. `showLanguageSwitch` 속성으로 제어하며, 활성화된 언어가 2개 이상일 때만 토글 가능하다. |

#### 4.8.2 선행 조건

- 설문에 다국어가 활성화되어 있어야 한다.
- `enabled=true`인 언어가 2개 이상이어야 한다.

#### 4.8.3 후행 조건

- `showLanguageSwitch=true`인 경우 응답자에게 언어 전환 UI가 표시된다.
- 응답자가 언어를 전환하면 설문이 선택한 언어로 재렌더링된다.

#### 4.8.4 기본 흐름

1. 설문 관리자가 Multi-Language Card에서 Language Switch 토글을 `true`로 설정한다.
2. 응답자가 설문에 접근한다.
3. 시스템이 활성화된 언어가 2개 이상인지 확인한다.
4. 조건 충족 시 설문 화면에 언어 전환 드롭다운/선택 UI를 표시한다.
5. 응답자가 다른 언어를 선택한다.
6. 시스템이 설문을 선택한 언어로 재렌더링한다.

#### 4.8.5 대안 흐름

- **AF-08-01**: `showLanguageSwitch=false`인 경우, 언어 전환 UI를 표시하지 않는다. 응답자는 초기 설정된 언어로만 응답한다.

#### 4.8.6 예외 흐름

- **EF-08-01**: 활성 언어가 1개 이하인 경우, Language Switch 토글이 비활성화된다 (설문 관리자가 토글을 켤 수 없다).

#### 4.8.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 동작 |
|---------|------|------|------|
| BR-08-01 | 최소 2개 활성 언어 | Language Switch 토글 시 | `enabled=true`인 언어가 2개 이상일 때만 토글 가능 |
| BR-08-02 | 토글 비활성화 | 활성 언어 1개 이하 | Language Switch 토글 UI를 비활성화(disabled) 상태로 표시 |
| BR-08-03 | 활성 언어만 노출 | 언어 전환 UI 표시 시 | `enabled=true`인 언어만 선택지에 표시 |

#### 4.8.8 데이터 요구사항

**입력 데이터**:

| 필드명 | 타입 | 필수 여부 | 설명 |
|--------|------|----------|------|
| showLanguageSwitch | boolean | 필수 | 언어 전환 UI 표시 여부 |

#### 4.8.9 화면/UI 요구사항

- **관리자 측**: Multi-Language Card에 Language Switch 토글 스위치
- **응답자 측**: 설문 화면 상단 또는 적절한 위치에 언어 선택 드롭다운
- 활성 언어 1개 이하인 경우 토글이 회색(disabled)으로 표시

#### 4.8.10 비기능 요구사항

- 없음

### 4.9 Link Survey 언어 지정

#### 4.9.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-015-09 |
| 기능명 | Link Survey 언어 지정 |
| 관련 요구사항 ID | FR-023 / 4.9 |
| 우선순위 | 필수 |
| 기능 설명 | Link Survey에서 `?lang=` URL 쿼리 파라미터를 통해 설문 표시 언어를 지정한다. 유효한 언어 코드가 전달되면 해당 언어로 설문을 표시하고, 무효한 코드이면 기본 언어로 fallback한다. |

#### 4.9.2 선행 조건

- 설문이 Link Survey 유형이어야 한다.
- 설문에 다국어가 활성화되어 있어야 한다.

#### 4.9.3 후행 조건

- 유효한 `?lang=` 파라미터: 해당 언어로 설문이 렌더링된다.
- 무효한 `?lang=` 파라미터: 기본 언어(`default`)로 설문이 렌더링된다.
- `?lang=` 파라미터 없음: 기본 언어로 설문이 렌더링된다.

#### 4.9.4 기본 흐름

1. 응답자가 Link Survey URL에 `?lang=ko`와 같은 쿼리 파라미터를 포함하여 접근한다.
2. 시스템이 `lang` 파라미터 값을 추출한다.
3. 시스템이 해당 언어 코드가 설문에서 `enabled=true`인 언어 중에 존재하는지 확인한다.
4. 해당 언어가 존재하고 활성화되어 있으면, 해당 언어로 설문을 렌더링한다.

#### 4.9.5 대안 흐름

- **AF-09-01 (lang 파라미터 없음)**: 기본 언어(`default=true`)로 설문을 렌더링한다.
- **AF-09-02 (다국어 미활성화 설문)**: `?lang=` 파라미터를 무시하고 기본 텍스트(`"default"` 키)로 렌더링한다.

#### 4.9.6 예외 흐름

- **EF-09-01**: 무효한 언어 코드(설문에 등록되지 않은 코드)가 전달된 경우, 기본 언어(`default`)로 fallback하여 렌더링한다.
- **EF-09-02**: 유효한 코드이나 `enabled=false`인 언어가 전달된 경우, 기본 언어로 fallback한다.

#### 4.9.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 동작 |
|---------|------|------|------|
| BR-09-01 | 유효한 코드 매칭 | `?lang=` 파라미터 수신 시 | 설문의 활성화된(`enabled=true`) 언어 코드와 매칭 |
| BR-09-02 | fallback 정책 | 무효한 코드 또는 파라미터 없음 | 기본 언어(`default=true`)로 렌더링 |
| BR-09-03 | URL 파라미터 형식 | 언어 지정 시 | `?lang=` 쿼리 파라미터 사용 (예: `?lang=ko`, `?lang=ar-SA`) |

#### 4.9.8 데이터 요구사항

**입력 데이터**:

| 필드명 | 타입 | 필수 여부 | 유효성 검증 규칙 |
|--------|------|----------|----------------|
| lang | string (쿼리 파라미터) | 선택 | ISO 언어 코드 형식, 설문의 활성 언어와 매칭 |

#### 4.9.9 화면/UI 요구사항

- 설문이 지정된 언어로 렌더링됨 (별도 UI 요소 없음)
- `showLanguageSwitch=true`인 경우 언어 전환 UI가 함께 표시되며, `?lang=` 파라미터로 지정된 언어가 초기 선택 상태

#### 4.9.10 비기능 요구사항

- 없음

### 4.10 RTL 언어 지원

#### 4.10.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-015-10 |
| 기능명 | RTL (Right-to-Left) 언어 지원 |
| 관련 요구사항 ID | FR-023 / 4.10 |
| 우선순위 | 필수 |
| 기능 설명 | RTL 언어(아랍어, 히브리어, 페르시아어, 우르두어 등)로 설문을 표시할 때 텍스트 방향과 레이아웃을 자동으로 RTL로 전환한다. |

#### 4.10.2 선행 조건

- 설문에 RTL 언어가 활성화되어 있거나, 단일 언어 설문에서 RTL 텍스트가 입력되어 있어야 한다.

#### 4.10.3 후행 조건

- RTL 언어로 설문이 표시될 때 `dir="rtl"` 속성이 적용된다.
- 로고 위치, 텍스트 정렬 등이 RTL 방향으로 전환된다.

#### 4.10.4 기본 흐름

**다국어 설정이 있는 경우**:
1. 시스템이 현재 표시 언어의 언어 코드를 확인한다.
2. 언어 코드에서 기본 코드(base code)를 추출한다 (예: `ar-SA` -> `ar`).
3. 기본 코드가 RTL 언어 목록에 포함되는지 확인한다.
4. RTL 언어인 경우 설문 컴포넌트 전체에 `dir="rtl"` 속성을 적용한다.
5. 로고 위치를 RTL 방향으로 조정한다.

**다국어 설정이 없는 경우**:
1. 시스템이 Welcome Card의 헤드라인 텍스트를 분석한다.
2. 유니코드 방향성(Bidi) 알고리즘으로 텍스트의 주요 방향을 판별한다.
3. RTL로 판별된 경우 `dir="rtl"` 속성을 적용한다.

#### 4.10.5 대안 흐름

- **AF-10-01**: LTR 언어인 경우 `dir="ltr"` 또는 기본 방향으로 설문을 렌더링한다 (별도 처리 불필요).

#### 4.10.6 예외 흐름

- 없음

#### 4.10.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 동작 |
|---------|------|------|------|
| BR-10-01 | RTL 언어 목록 | RTL 판별 시 | `ar`, `ar-SA`, `ar-EG`, `ar-AE`, `ar-MA`, `he`, `fa`, `ur` (총 8개) |
| BR-10-02 | base code 추출 | 언어 코드 판별 시 | 하이픈(`-`) 이전의 기본 코드로 RTL 여부 판별 (예: `ar-SA` -> `ar`) |
| BR-10-03 | 다국어 우선 | RTL 판별 로직 | 다국어 설정이 있으면 언어 코드 기반, 없으면 유니코드 방향성 기반 |

**RTL 지원 언어 전체 목록**:

| 코드 | 언어 | base code |
|------|------|-----------|
| ar | 아랍어 (일반) | ar |
| ar-SA | 아랍어 (사우디아라비아) | ar |
| ar-EG | 아랍어 (이집트) | ar |
| ar-AE | 아랍어 (UAE) | ar |
| ar-MA | 아랍어 (모로코) | ar |
| he | 히브리어 | he |
| fa | 페르시아어 | fa |
| ur | 우르두어 | ur |

#### 4.10.8 데이터 요구사항

**입력 데이터**:

| 필드명 | 타입 | 필수 여부 | 설명 |
|--------|------|----------|------|
| languageCode | string | 조건부 (다국어 활성 시) | 현재 표시 언어의 ISO 코드 |
| welcomeCardHeadline | string | 조건부 (단일 언어 시) | Welcome Card 헤드라인 텍스트 |

**출력 데이터**:

| 필드명 | 타입 | 설명 |
|--------|------|------|
| isRTL | boolean | RTL 여부 판별 결과 |
| dir | "rtl" \| "ltr" | HTML dir 속성 값 |

#### 4.10.9 화면/UI 요구사항

- **RTL 적용 시**: 설문 컴포넌트 전체에 `dir="rtl"` 속성 적용
- **로고 위치**: RTL 시 로고 위치가 반대 방향으로 이동 (예: 좌측 -> 우측)
- **텍스트 정렬**: RTL 방향에 맞게 자동 조정

#### 4.10.10 비기능 요구사항

- NFR-015-03: RTL 언어 자동 감지 (언어 코드 또는 텍스트 방향성 기반)

### 4.13 텍스트 필드 언어별 검증

#### 4.13.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-015-13 |
| 기능명 | 텍스트 필드 언어별 검증 |

| 기능 설명 | 활성화된 모든 언어에 대해 텍스트 필드가 비어있지 않은지 검증한다. HTML 콘텐츠인 경우 태그를 제거한 실제 텍스트를 기준으로 검증한다. |

#### 4.13.2 선행 조건

- 설문에 다국어가 활성화되어 있어야 한다.
- 1개 이상의 언어가 `enabled=true`로 활성화되어 있어야 한다.

#### 4.13.3 후행 조건

- 모든 활성 언어에 대해 번역이 완료된 경우: 검증 통과.
- 번역이 누락된 언어가 있는 경우: 검증 경고/오류가 반환된다.

#### 4.13.4 기본 흐름

1. 설문 저장 또는 발행 시 시스템이 언어별 텍스트 검증을 수행한다.
2. 시스템이 설문의 모든 TI18nString 필드를 탐색한다.
3. 각 필드에 대해 `enabled=true`인 모든 언어의 텍스트가 존재하는지 확인한다.
4. 텍스트가 HTML인 경우, HTML 태그를 제거한 후 실제 텍스트가 비어있지 않은지 확인한다.
5. 모든 필드에서 모든 활성 언어의 텍스트가 존재하면 검증 통과.

#### 4.13.5 대안 흐름

- 없음

## 5. 데이터 모델

### 5.1 주요 엔티티 정의

#### 5.1.1 Language (프로젝트 레벨 언어)

| 필드명 | 타입 | 필수 | 기본값 | 설명 |
|--------|------|------|--------|------|
| id | string (CUID2) | 필수 | 자동 생성 | 언어 고유 식별자 |
| code | string | 필수 | - | ISO 언어 코드 (예: `en`, `ko`, `ar-SA`) |
| alias | string \| null | 선택 | null | 사용자 정의 별칭 (표시용) |
| projectId | string (CUID2) | 필수 | - | 소속 프로젝트 ID |
| createdAt | DateTime | 필수 | 자동 생성 | 생성 일시 |
| updatedAt | DateTime | 필수 | 자동 갱신 | 수정 일시 |

#### 5.1.2 SurveyLanguage (설문 레벨 언어 설정)

| 필드명 | 타입 | 필수 | 기본값 | 설명 |
|--------|------|------|--------|------|
| surveyId | string (CUID2) | 필수 | - | 소속 설문 ID |
| languageId | string (CUID2) | 필수 | - | 참조 Language ID |
| default | boolean | 필수 | false | 기본 언어 여부 (설문당 1개만 true) |
| enabled | boolean | 필수 | true | 활성화 여부 |

#### 5.1.3 TI18nString (다국어 문자열 타입)

| 키 | 타입 | 필수 | 설명 |
|----|------|------|------|
| "default" | string | 필수 | 기본 언어 텍스트 |
| [ISO 코드] | string | 선택 | 해당 언어의 번역 텍스트 |

### 5.2 엔티티 간 관계

```
Project (1) ----< (N) Language
    |
    +---< (N) Survey (1) ----< (N) SurveyLanguage (N) >---- (1) Language
                   |
                   +--- TI18nString 필드들 (질문 텍스트, 설명, 선택지 등)
```

- **Project : Language** = 1:N (하나의 프로젝트에 여러 언어 등록 가능)
- **Survey : SurveyLanguage** = 1:N (하나의 설문에 여러 설문 언어 설정)
- **Language : SurveyLanguage** = 1:N (하나의 언어가 여러 설문에서 참조)
- **Survey : TI18nString** = 1:N (하나의 설문에 여러 다국어 텍스트 필드)

### 5.3 데이터 흐름

```
[프로젝트 설정]              [설문 에디터]              [설문 런타임]
      |                          |                          |
  언어 CRUD                 언어 활성화              ?lang= 파라미터
      |                    번역 텍스트 입력                  |
      v                          v                          v
  Language 테이블     SurveyLanguage 테이블          TI18nString 렌더링
      |               + TI18nString 필드                    |
      |                          |                   RTL 판별 + 적용
      +---------> 참조 <---------+                          |
                                                  Language Switch UI
```

### 6.1 외부 시스템 연동

### 6.2 API 명세 (해당 시)

#### 6.2.1 프로젝트 언어 관리 API

**언어 목록 조회**:
- 엔드포인트: `GET /api/v1/management/projects/{projectId}/languages`
- 인증: API Key (Bearer token)
- 응답: `Language[]`

**언어 생성**:
- 엔드포인트: `POST /api/v1/management/projects/{projectId}/languages`
- 인증: API Key (Bearer token)
- 요청 본문: `{ code: string, alias?: string }`
- 응답: `Language`

**언어 수정**:
- 엔드포인트: `PUT /api/v1/management/projects/{projectId}/languages/{languageId}`
- 인증: API Key (Bearer token)
- 요청 본문: `{ code?: string, alias?: string | null }`
- 응답: `Language`

**언어 삭제**:
- 엔드포인트: `DELETE /api/v1/management/projects/{projectId}/languages/{languageId}`
- 인증: API Key (Bearer token)
- 응답: `{ success: boolean }`

#### 6.2.2 설문 언어 설정 API

설문 CRUD API 내에서 `languages` 필드를 통해 관리한다.

- 설문 생성/수정 시 `languages: SurveyLanguage[]` 필드에 언어 설정을 포함한다.
- 설문 조회 시 `languages` 필드에 설정된 언어 목록이 반환된다.

#### 6.2.3 Link Survey 언어 지정

- URL 형식: `https://{domain}/s/{surveyId}?lang={languageCode}`
- 파라미터: `lang` (선택, ISO 언어 코드)
- 동작: 유효한 코드 -> 해당 언어로 렌더링, 무효한 코드 -> 기본 언어 fallback
