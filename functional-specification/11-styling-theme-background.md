## 2. 개요

### 2.1 목적

Inquiry 설문 UI의 시각적 커스터마이징을 위한 계층적 스타일링 시스템의 상세 기능을 정의한다. Project(프로젝트) 레벨에서 전체 테마를 설정하고, 개별 Survey(설문) 레벨에서 해당 테마를 오버라이드하는 구조를 통해 브랜드 일관성과 개별 설문 디자인 독립성을 동시에 달성한다.

### 2.2 범위

**In-scope**:
- Project 레벨 기본 테마 설정 (프로젝트 스타일링 구조)
- Survey 레벨 스타일 오버라이드 (설문 스타일링 구조)
- 배경 4가지 타입: color, image, upload, animation
- Dark Mode 토글
- 카드 배치(Card Arrangement) 설정
- 로고 설정 및 숨김
- 프로그레스바 커스터마이징
- Brand Color 기반 자동 색상 생성(Suggest Colors)
- Legacy 스타일 필드(v4.6)에서 신규 필드(v4.7)로의 자동 마이그레이션

**Out-of-scope**:
- CSS 직접 편집
- 커스텀 폰트 업로드 (폰트 패밀리는 문자열 지정만 가능)
- Survey 런타임에서의 동적 스타일 변경

### 2.3 대상 사용자

| 역할 | 관심 사항 |
|------|----------|
| 설문 작성자 | 브랜드에 맞는 시각적 커스터마이징 |
| 프로젝트 관리자 | 전체 프로젝트의 일관된 테마 관리 |
| 응답자 | 가독성 높고 시각적으로 쾌적한 설문 경험 |
| 프론트엔드 개발자 | 스타일링 타입과 컴포넌트 구조 이해 |

### 2.4 용어 정의

| 용어 | 정의 |
|------|------|
| Project 스타일링 | 프로젝트 전체에 적용되는 기본 테마 설정. Look & Feel 페이지에서 관리한다. |
| Survey 스타일링 | 개별 설문에 적용되는 스타일 오버라이드. 설문 에디터의 Styling 탭에서 관리한다. |
| 테마 오버라이드 | Survey 레벨에서 Project 테마를 무시하고 독자적 스타일을 적용하는 기능이다. |
| Brand Color | 브랜드 기본 색상. Suggest Colors 기능의 기준점이 된다. |
| Dark Mode | 설문의 어두운 테마 모드. 활성화 시 각 색상 속성의 dark 값이 적용된다. |
| Card Arrangement | 설문 카드의 배치 스타일. casual, straight, simple 중 선택한다. |
| Suggest Colors | Brand Color 하나에서 전체 색상 팔레트를 자동 생성하는 기능이다. |
| Legacy 필드 마이그레이션 | v4.6 스타일링 필드를 v4.7 세분화된 필드로 자동 변환하는 처리이다. |
| 색상(light/dark) | light(필수)와 dark(선택) 두 값을 가지는 색상 타입이다. |
| Link Survey | 독립 URL로 접근하는 설문 유형. 배경 및 로고 설정이 가능하다. |
| App Survey | 인앱 임베드 방식의 설문 유형. 배경 및 로고 설정이 불가하다. |

## 4. 상세 기능 명세

### 4.1 기반 스타일링 속성 관리 (FN-011-01)

#### 4.1.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-011-01 |
| 기능명 | 기반 스타일링 속성 관리 |
| 관련 요구사항 ID | FR-015 |
| 우선순위 | 필수 |
| 기능 설명 | 설문 UI를 구성하는 40개 이상의 스타일링 속성을 정의하고 관리한다. 색상, 타이포그래피, 버튼, 인풋, 옵션, 프로그레스바, 카드/레이아웃 카테고리로 분류된다. |

#### 4.1.2 선행 조건

- 사용자가 프로젝트에 접근 권한을 보유하고 있어야 한다.
- 스타일링 설정 페이지(Project Look & Feel 또는 Survey Editor Styling 탭)에 접근한 상태여야 한다.

#### 4.1.3 후행 조건

- 변경된 스타일링 속성이 저장되어 설문 렌더링에 반영된다.
- 스키마 유효성 검증을 통과한 데이터만 저장된다.

#### 4.1.4 기본 흐름

1. 사용자가 스타일링 설정 페이지에 접근한다.
2. 시스템이 현재 적용된 스타일링 속성 값을 로드하여 폼에 표시한다.
3. 사용자가 원하는 속성 값을 변경한다.
4. 시스템이 변경된 값을 실시간으로 설문 미리보기에 반영한다 (NFR-011-01).
5. 사용자가 저장 버튼을 클릭한다.
6. 시스템이 스키마 유효성 검증을 수행한다.
7. 검증 통과 시 변경 사항을 저장한다.

#### 4.1.5 대안 흐름

- **AF-01**: 사용자가 값을 변경하지 않고 페이지를 떠나는 경우, 변경 사항 없이 기존 값이 유지된다.
- **AF-02**: 사용자가 속성 값을 비우는 경우(null/undefined), 상위 레벨의 기본값으로 fallback된다 (NFR-011-02).

#### 4.1.6 예외 흐름

- **EF-01**: 스키마 유효성 검증 실패 시, 시스템이 해당 필드에 에러를 표시하고 저장을 중단한다.
- **EF-02**: 색상 값이 유효한 CSS Color 포맷이 아닌 경우, 유효성 검증 에러를 표시한다 (NFR-011-03).

#### 4.1.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 액션 |
|---------|------|------|------|
| BR-01-01 | 모든 색상 속성은 light/dark 쌍으로 구성된다 | 색상 타입 속성 | light는 필수, dark는 선택 |
| BR-01-02 | null/undefined 값은 상위 레벨로 fallback된다 | 속성 값이 null 또는 undefined | 시스템 기본값 -> Project -> Survey 순으로 해석 |
| BR-01-03 | 모든 색상은 유효한 CSS Color여야 한다 | 색상 값 저장 시 | CSS Color 포맷 유효성 검증 수행 |
| BR-01-04 | 폰트 패밀리는 문자열로만 지정 가능하다 | 폰트 패밀리 설정 시 | 문자열 입력만 허용, 파일 업로드 불가 |

#### 4.1.9 화면/UI 요구사항

- 각 속성 카테고리(General Colors, Headlines, Buttons, Inputs, Options, Progress Bar, Card & Layout)는 접을 수 있는(collapsible) 섹션으로 구성한다.
- 색상 속성은 컬러 피커(color picker)로 입력을 받는다.
- 숫자 속성은 숫자 입력 필드 또는 슬라이더로 입력을 받는다.
- 모든 변경 사항은 실시간으로 설문 미리보기 영역에 반영되어야 한다 (NFR-011-01).

#### 4.1.10 비기능 요구사항

- 스타일 변경 시 실시간 미리보기 반영 (폼 값 변경 구독으로 즉시 반영) (NFR-011-01).
- null/undefined 값은 상위 레벨 기본값으로 fallback (선택적 값 스키마로 처리) (NFR-011-02).

### 4.4 배경 설정 (FN-011-04)

#### 4.4.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-011-04 |
| 기능명 | 배경 설정 (Background) |
| 관련 요구사항 ID | FR-016 |
| 우선순위 | 필수 |
| 기능 설명 | 설문 배경을 4가지 타입(color, image, upload, animation) 중 하나로 설정하고 밝기를 조절한다. Link Survey에서만 설정 가능하다. |

#### 4.4.2 선행 조건

- 스타일링 설정 페이지에 접근한 상태여야 한다.
- 설문 타입이 Link Survey여야 한다 (App Survey에서는 배경 설정 섹션이 노출되지 않는다).

#### 4.4.3 후행 조건

- 선택한 배경 타입과 값, 밝기 설정이 저장된다.
- upload 타입의 경우 업로드된 이미지 URL이 저장된다.

#### 4.4.4 기본 흐름

**color 타입**:
1. 사용자가 배경 타입으로 "color"를 선택한다.
2. 컬러 피커가 표시된다.
3. 사용자가 CSS 색상 코드를 입력하거나 피커에서 선택한다.
4. 밝기 슬라이더로 밝기를 조절한다 (기본값: 100).
5. 미리보기에 선택한 단색 배경이 즉시 반영된다.

**image 타입**:
1. 사용자가 배경 타입으로 "image"를 선택한다.
2. Unsplash 이미지 라이브러리 브라우저가 표시된다.
3. 사용자가 원하는 이미지를 검색하고 선택한다.
4. 선택한 이미지의 Unsplash URL이 bg 값으로 설정된다.
5. 밝기 슬라이더로 밝기를 조절한다.
6. 미리보기에 선택한 이미지 배경이 즉시 반영된다.

**upload 타입**:
1. 사용자가 배경 타입으로 "upload"를 선택한다.
2. 이미지 업로드 영역이 표시된다.
3. 사용자가 이미지 파일을 업로드한다.
4. 업로드된 이미지의 URL이 bg 값으로 설정된다.
5. 밝기 슬라이더로 밝기를 조절한다.
6. 미리보기에 업로드한 이미지 배경이 즉시 반영된다.

**animation 타입**:
1. 사용자가 배경 타입으로 "animation"를 선택한다.
2. 사용 가능한 애니메이션 목록이 표시된다.
3. 사용자가 원하는 애니메이션을 선택한다.
4. 선택한 애니메이션 식별자가 bg 값으로 설정된다.
5. 밝기 슬라이더로 밝기를 조절한다.
6. 미리보기에 선택한 동적 배경이 즉시 반영된다.

#### 4.4.5 대안 흐름

- **AF-01**: Unsplash 연동이 비활성화된 경우, image 타입 옵션이 노출되지 않는다.
- **AF-02**: 스토리지 설정이 비활성화된 경우, upload 타입 옵션이 노출되지 않는다.

#### 4.4.6 예외 흐름

- **EF-01**: upload 타입에서 bg 값(이미지 URL)이 없는 상태로 저장을 시도하면, 유효성 검증 실패 에러가 발생한다 (AC-016-02).
- **EF-02**: App Survey에서 배경 설정 섹션에 접근을 시도하면, 해당 섹션이 숨겨져 있어 접근할 수 없다 (AC-017-04).

#### 4.4.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 액션 |
|---------|------|------|------|
| BR-04-01 | 배경 타입은 4가지 중 하나여야 한다 | 배경 설정 시 | color, image, upload, animation 중 택 1 |
| BR-04-02 | upload 타입에서 bg 값은 필수이다 | bgType=upload 저장 시 | bg 값이 없으면 유효성 검증 실패 |
| BR-04-03 | 밝기 기본값은 100이다 | 배경 설정 초기화 시 | brightness = 100 |
| BR-04-04 | 배경 설정은 Link Survey에서만 가능하다 | Survey 타입 확인 | App Survey에서는 배경 설정 섹션 숨김 |
| BR-04-05 | 배경 타입 활성화는 외부 설정에 의존한다 | 배경 타입 선택 시 | Unsplash 연동 여부, 스토리지 설정 여부 플래그로 결정 |

#### 4.4.8 데이터 요구사항

**입력 데이터**

| 필드명 | 타입 | 필수 | 유효성 규칙 | 기본값 |
|--------|------|------|------------|--------|
| bgType | enum | 선택 | "color" / "image" / "upload" / "animation" | - |
| bg | string | 조건부 필수 | bgType=color: CSS 색상 코드, bgType=image: Unsplash URL, bgType=upload: 업로드된 이미지 URL (필수), bgType=animation: 애니메이션 식별자 | - |
| brightness | number | 선택 | 양수 | 100 |

**배경 타입별 bg 값 형식**

| 타입 | bg 값 형식 | 예시 |
|------|-----------|------|
| color | CSS 색상 코드 | #ffffff |
| image | Unsplash 이미지 URL | https://images.unsplash.com/... |
| upload | 업로드된 이미지 URL | /storage/backgrounds/abc123.jpg |
| animation | 애니메이션 식별자 | starfield |

7. 사용자가 저장한다.

#### 4.6.5 대안 흐름

- **AF-01**: 사용자가 토글을 비활성화(false)하면, 설문 자체 스타일 속성이 무시되고 프로젝트 테마가 적용된다.

- **AF-03**: 프로젝트의 "스타일 오버라이드 허용"이 false인 경우, 이 토글 자체가 비활성화되어 사용자가 오버라이드를 설정할 수 없다.

#### 4.6.6 예외 흐름

- 없음.

#### 4.6.7 비즈니스 규칙

### 4.7 스타일 적용 우선순위 해석 (FN-011-07)

#### 4.7.1 기능 개요

| 항목 | 내용 |

| 우선순위 | 필수 |
| 기능 설명 | 설문 렌더링 시 시스템 기본값, 프로젝트 스타일링, 설문 스타일링 간의 우선순위를 결정하여 최종 스타일을 계산한다. Legacy 필드 마이그레이션도 이 과정에서 처리된다. |

#### 4.7.2 선행 조건

- 설문 렌더링 요청이 발생하여야 한다.

#### 4.7.3 후행 조건

- 최종 스타일 객체가 생성되어 설문 렌더링에 사용된다.

#### 4.7.4 기본 흐름

4. 프로젝트 스타일링 설정 값으로 기반 스타일을 오버라이드한다.
5. 설문의 overrideTheme 값을 확인한다.
6. overrideTheme = true인 경우:
   a. 설문 스타일링 데이터를 로드한다.
   b. 설문 레거시 필드가 존재하면 마이그레이션을 수행한다.
   c. 설문 스타일링 설정 값으로 기반 스타일을 오버라이드한다.
7. 최종 스타일 객체를 반환한다.

#### 4.7.5 대안 흐름

- **AF-01**: overrideTheme = false인 경우, 5단계에서 프로젝트 스타일이 최종 스타일로 확정된다 (6, 7단계의 설문 스타일 오버라이드 생략).

#### 4.7.6 예외 흐름

- 없음. 모든 단계에서 null/undefined 값은 이전 단계의 값을 유지한다.

### 4.8 Suggest Colors - 자동 색상 생성 (FN-011-08)

#### 4.8.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-011-08 |
| 기능명 | Suggest Colors (자동 색상 생성) |
| 관련 요구사항 ID | FR-017 |
| 우선순위 | 필수 |
| 기능 설명 | 단일 Brand Color에서 전체 색상 팔레트를 자동으로 생성하여 조화로운 설문 디자인을 쉽게 구성할 수 있도록 한다. |

#### 4.8.2 선행 조건

- Brand Color가 설정되어 있어야 한다.
- 스타일링 설정 페이지에 접근한 상태여야 한다.

#### 4.8.3 후행 조건

- Brand Color를 기반으로 생성된 색상 팔레트가 해당 스타일링 속성에 적용된다 (AC-017-01).
- 사용자가 저장하기 전까지 실시간 미리보기에만 반영된다.

#### 4.8.4 기본 흐름

1. 사용자가 Brand Color를 설정한다.
2. 사용자가 "Suggest Colors" 버튼을 클릭한다.
3. 시스템이 Brand Color를 기반으로 다음 색상을 자동 생성한다:
   - 질문 텍스트 색상: Brand Color + #000000 (35% 혼합) -> 어두운 텍스트 색상
   - 인풋 배경: Brand Color + #ffffff (92% 혼합) -> 밝은 인풋 배경
   - 인풋 테두리: Brand Color + #ffffff (60% 혼합) -> 중간 테두리
   - 카드 배경: Brand Color + #ffffff (97% 혼합) -> 거의 흰색 카드 배경
   - 페이지 배경: Brand Color + #ffffff (85.5% 혼합) -> 페이지 배경
   - 버튼 텍스트: Brand Color 밝기에 따라 자동 결정
4. 생성된 색상 값이 각 필드에 자동 입력된다.
5. 미리보기에 변경된 색상 팔레트가 즉시 반영된다.

#### 4.8.5 대안 흐름

- **AF-01**: 사용자가 생성된 색상 중 일부를 수동으로 변경할 수 있다. 수동 변경 후에는 해당 필드만 사용자 지정 값으로 유지된다.

#### 4.8.6 예외 흐름

- **EF-01**: Brand Color가 미설정 상태에서 "Suggest Colors" 버튼 클릭 시, Brand Color 설정을 먼저 요구하는 안내를 표시한다.

#### 4.8.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 액션 |

### 4.9 Legacy 필드 마이그레이션 (FN-011-09)

#### 4.9.1 기능 개요

| 항목 | 내용 |
|------|------|
| 기능 ID | FN-011-09 |
| 기능명 | Legacy 필드 마이그레이션 |
| 관련 요구사항 ID | FR-017 |
| 우선순위 | 필수 |
| 기능 설명 | v4.6의 레거시 스타일링 필드를 v4.7의 세분화된 신규 필드로 자동 변환한다. 스타일 해석 과정에서 자동으로 수행된다. |

#### 4.9.2 선행 조건

- v4.6 형식의 레거시 스타일링 필드가 포함된 데이터가 로드되어야 한다.

#### 4.9.3 후행 조건

- 레거시 필드 값이 v4.7 신규 필드로 변환되어 적용된다 (AC-017-03).
- 신규 필드에 이미 값이 존재하는 경우, 신규 필드 값이 우선한다.

#### 4.9.4 기본 흐름

1. 시스템이 스타일링 데이터를 로드한다.
2. 레거시 필드(질문 텍스트 색상, 브랜드 색상, 인풋 배경 색상) 존재 여부를 확인한다.
3. 레거시 필드가 존재하면, 매핑 규칙에 따라 신규 필드에 값을 할당한다.
4. 신규 필드에 이미 명시적 값이 설정되어 있으면, 해당 필드는 마이그레이션을 건너뛴다.
5. 변환된 스타일 데이터를 반환한다.

#### 4.9.5 대안 흐름

- **AF-01**: 레거시 필드가 존재하지 않으면, 마이그레이션을 수행하지 않고 원본 데이터를 그대로 사용한다.

#### 4.9.6 예외 흐름

- 없음. 마이그레이션은 데이터 변환 로직으로서 에러 상황이 발생하지 않는다.

#### 4.9.7 비즈니스 규칙

| 규칙 ID | 규칙 | 조건 | 액션 |
|---------|------|------|------|
| BR-09-01 | 신규 필드가 레거시 마이그레이션 값보다 우선한다 | 신규 필드에 값이 이미 존재 | 마이그레이션 건너뜀 |
| BR-09-02 | Project와 Survey 레벨 모두에서 독립적으로 마이그레이션을 수행한다 | 스타일 해석 과정 | 각 레벨의 레거시 필드를 별도 처리 |

#### 4.9.8 데이터 요구사항

**입력 데이터**

4. 로고 배경색을 설정한다.
5. 필요 시 "로고 숨김" 토글로 로고를 숨긴다.
6. 미리보기에 로고 변경이 즉시 반영된다.
7. 사용자가 저장한다.

#### 4.10.5 대안 흐름

- **AF-01**: App Survey에서는 로고 설정 섹션이 숨겨진다 (AC-017-04).
- **AF-02**: 로고 숨김을 활성화하면 로고 URL 및 배경색 설정은 유지되지만 렌더링 시 표시되지 않는다.

#### 4.10.6 예외 흐름

#### 4.11.2 선행 조건

- 스타일링 설정 페이지에 접근한 상태여야 한다.

#### 4.11.3 후행 조건

- 프로그레스바 스타일 설정이 저장되어 설문 렌더링에 반영된다.

1. 사용자가 스타일링 설정의 Progress Bar 섹션에 접근한다.

3. 사용자가 원하는 값으로 변경한다.
4. 미리보기에 변경이 즉시 반영된다.
5. 사용자가 저장한다.

#### 4.11.5 대안 흐름

- **AF-01**: "프로그레스바 숨김"을 활성화하면, 프로그레스바가 설문에서 표시되지 않는다.
