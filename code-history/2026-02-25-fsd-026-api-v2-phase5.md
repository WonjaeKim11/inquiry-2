# FSD-026 Contact Management - Phase 5: API v2 (Bulk Upload + SDK Identify)

## Overview
Contact 관리 시스템에 v2 API 엔드포인트를 추가한다. Management API를 통한 Bulk Upload(최대 250건 일괄 생성)와 Client API를 통한 SDK Identify(userId 기반 연락처 자동 생성/업데이트) 기능을 제공한다. 기존 v1 API와 독립적으로 v2 경로에 매핑되어 하위 호환성을 유지한다.

## Major Changes

### 1. Bulk Upload API (Management API v2)

`ContactManagementApiController`는 API Key 인증(`ApiKeyAuthGuard`) + 권한 검증(`RequirePermissionGuard`) + 속도 제한(`ApiRateLimitGuard`)을 적용하며, `@Controller('v2/management/contacts')` 절대 경로로 RouterModule 수정 없이 자동 매핑된다.

```typescript
// POST /v2/management/contacts/bulk?environmentId=xxx
@Post('bulk')
@RequirePermission('WRITE')
@HttpCode(HttpStatus.CREATED)
async bulkUpload(
  @Query('environmentId') environmentId: string,
  @Body(new ZodValidationPipe(BulkUploadSchema)) dto: BulkUploadDto
) { ... }
```

처리 흐름:
1. `seedDefaultKeys`로 기본 속성 키 시딩 (멱등)
2. `getKeyMap`으로 환경의 전체 속성 키 맵 로드
3. 각 연락처 항목에 대해:
   - `userId`가 있으면 → `identifyByUserId()`로 upsert (기존 연락처 업데이트 또는 새로 생성)
   - `userId`가 없으면 → `createContactWithAttributes()`로 새 연락처 + 속성 직접 생성
4. 개별 레코드 실패 시 `errors` 카운트 증가, 나머지 레코드는 계속 처리

`createContactWithAttributes` 내부에서 미등록 속성 키는 `TypeDetectorService.detectType()`으로 타입을 추론하여 자동 생성한다.

### 2. SDK Identify API (Client API v2)

`ContactClientApiController`는 `EnvironmentIdGuard`로 환경 ID 유효성을 검증하며, `@Controller('v2/client/:environmentId/contacts')` 절대 경로로 매핑된다.

```typescript
// POST /v2/client/:environmentId/contacts/identify
@Post('identify')
@HttpCode(HttpStatus.OK)
async identify(
  @Param('environmentId') environmentId: string,
  @Body(new ZodValidationPipe(SdkIdentifySchema)) dto: SdkIdentifyDto
) { ... }
```

기존 `ContactService.identifyByUserId()`를 그대로 호출하여 userId 기반 연락처 생성/업데이트를 수행한다.

### 3. 절대 경로 라우팅 전략

`ContactModule`은 단일 모듈이므로 `RouterModule.register()`에 별도 항목을 추가하면 내부 컨트롤러(ContactController, ContactAttributeController)도 해당 경로에 노출되는 문제가 있다. 이를 해결하기 위해 v2 컨트롤러는 `@Controller()` 데코레이터에 절대 경로를 사용한다:
- `v2/management/contacts` (Management API)
- `v2/client/:environmentId/contacts` (Client API)

## How to use it

### Bulk Upload (Management API)
```bash
curl -X POST "https://api.example.com/api/v2/management/contacts/bulk?environmentId=ENV_ID" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "contacts": [
      { "userId": "user-1", "email": "a@b.com", "attributes": { "firstName": "Alice" } },
      { "userId": "user-2", "attributes": { "plan": "pro" } },
      { "email": "c@d.com", "attributes": { "source": "import" } }
    ]
  }'

# Response (201 Created):
# { "data": { "created": 2, "updated": 1, "errors": 0 } }
```

### SDK Identify (Client API)
```bash
curl -X POST "https://api.example.com/api/v2/client/ENV_ID/contacts/identify" \
  -H "Content-Type: application/json" \
  -d '{
    "userId": "sdk-user-123",
    "attributes": { "email": "user@example.com", "plan": "enterprise" }
  }'

# Response (200 OK):
# { "data": { "contactId": "cuid2...", "isNew": true } }
```

### 단건 조회 / 삭제
```bash
# 조회
curl "https://api.example.com/api/v2/management/contacts/CONTACT_ID?environmentId=ENV_ID" \
  -H "x-api-key: YOUR_API_KEY"

# 삭제
curl -X DELETE "https://api.example.com/api/v2/management/contacts/CONTACT_ID?environmentId=ENV_ID" \
  -H "x-api-key: YOUR_API_KEY"
```

## Related Components/Modules
- `ContactService.identifyByUserId()` - Bulk Upload와 SDK Identify 모두에서 사용하는 핵심 비즈니스 로직
- `ContactAttributeService` - 속성 키 시딩, 키 맵 조회, 속성 값 upsert
- `TypeDetectorService` - 미등록 속성 키의 데이터 타입 자동 추론
- `ManagementContactController` (v1) - 기존 v1 Management API (하위 호환성 유지)
- `EnvironmentIdGuard` (`@inquiry/server-client-api`) - Client API 환경 ID 검증
- `ApiKeyAuthGuard`, `RequirePermissionGuard` (`@inquiry/server-api-key`) - Management API 인증/인가
- `ApiRateLimitGuard` (`@inquiry/server-rate-limit`) - API 속도 제한

## Precautions
- Bulk Upload의 최대 건수는 `MAX_BULK_SIZE=250`으로 제한된다. 더 큰 데이터셋은 CSV Import(`POST /environments/:envId/contacts/import`)를 사용해야 한다.
- Bulk Upload에서 개별 레코드 실패는 전체 요청을 중단시키지 않으며, `errors` 카운트로 반환된다. 구체적인 실패 항목 인덱스는 현재 응답에 포함되지 않는다.
- `createContactWithAttributes`에서 `ServerPrismaService`를 직접 DI로 주입받아 사용한다 (서비스 레이어 우회).
- v2 컨트롤러는 절대 경로를 사용하므로, 글로벌 API prefix(`/api`)가 설정되어 있다면 최종 경로는 `/api/v2/management/contacts/...`가 된다.
