# 다국어 설문 — 요구사항 명세서

> **문서번호**: FSD-015 | **FR 범위**: FR-023
> **라이선스**: Enterprise (다국어 설문 기능 플래그 필요)

---

## 1. 목적/배경

다국어 설문(Multi-Language Surveys)은 하나의 설문을 여러 언어로 번역하여 글로벌 대상으로 배포할 수 있게 해주는 Enterprise 기능이다. 프로젝트 레벨에서 사용 가능한 언어를 설정하고, 개별 설문에서 해당 언어들을 활성화하여 번역을 관리한다.

다국어 문자열 타입을 통해 모든 텍스트 필드가 다국어를 지원하며, "default" 키를 기본 언어로 사용한다. Link Survey에서는 ?lang= 쿼리 파라미터로 언어를 지정할 수 있고, RTL(Right-to-Left) 언어도 지원한다.

---

## 2. 범위 (In-scope / Out-of-scope)

### In-scope
- 다국어 문자열 타입 및 "default" 키 요구사항
- 프로젝트 레벨 언어 설정 (언어 구조)
- 설문 레벨 언어 활성화 (설문 언어 구조)
- 기본 언어(default) 및 보조 언어(secondary) 관리
- ?lang= 쿼리 파라미터를 통한 언어 지정
- RTL 언어 지원
- 언어 전환 UI (showLanguageSwitch)
- Enterprise 라이선스 검증 (다국어 설문 기능 플래그)

### Out-of-scope
- 자동 번역 (AI 기반 자동 번역 기능)
- 번역 워크플로우 / 번역자 역할 관리
- i18n 관련 관리자 UI (앱 자체의 UI 언어 설정 -- 별도 i18n 시스템)

## 4. 기능 요구사항

### FR-023: 다국어 설문 시스템

#### 4.1 다국어 문자열 타입 (TI18nString)

**구조**: 키-값 쌍의 레코드 형태 (Record<string, string>)

사용 예시:
- default: "What is your name?" (기본 언어, 필수)
- de: "Wie heissen Sie?" (독일어)
- ko: "이름이 무엇인가요?" (한국어)
- ar: "ما اسمك؟" (아랍어, RTL)

**필수 규칙**: "default" 키가 반드시 존재해야 함 (유효성 검증)

#### 4.2 프로젝트 레벨 언어 설정

**언어 속성**:

| 속성 | 타입 | 설명 |
|------|------|------|
| id | 문자열 (CUID2) | 언어 고유 식별자 |
| code | 문자열 | ISO 언어 코드 (예: en, ko, ar-SA) |
| alias | 문자열 또는 null | 사용자 정의 별칭 (표시용) |
| projectId | 문자열 (CUID2) | 소속 프로젝트 ID |

- 언어는 프로젝트 레벨에서 관리되며, 모든 설문에서 공유
- 프로젝트 설정 > Languages 페이지에서 CRUD 가능

#### 4.3 설문 레벨 언어 활성화

설문 언어 구조:
- language: 프로젝트에 등록된 언어 참조
- default: 기본 언어 여부 (boolean)
- enabled: 이 설문에서 활성화 여부 (boolean)

- 설문에는 언어 배열로 관리
- 정확히 하나의 언어가 default=true여야 함
- enabled=true인 언어만 응답자에게 제공됨

#### 4.4 Multi-Language Card UI

**활성화 조건**:
- Enterprise 라이선스의 다국어 설문 기능이 활성화되어야 함
- 프로젝트에 최소 2개 이상의 언어가 등록되어야 다국어 활성화 가능

**UI 동작 흐름**:

1. **라이선스 미보유 시**: UpgradePrompt 표시 (Cloud: 무료 체험 시작, Self-hosted: 체험 라이선스 요청)
2. **언어 미등록 시**: "No languages found" 메시지 + 언어 관리 페이지 링크
3. **언어 1개만 등록 시**: "2개 이상 언어 필요" 안내 메시지
4. **정상 활성화 시**:
   - Default Language 선택 드롭다운
   - Secondary Language 선택/토글
   - Language Switch 토글 옵션

#### 4.5 기본 언어 변경

- 기존 기본 언어의 default를 false로 변경
- 새 언어의 default를 true로 설정
- 해당 언어가 설문에 없으면 자동으로 추가

#### 4.6 보조 언어 추가/제거

- 이미 있는 언어면 enabled 토글
- 없는 언어면 새로 추가 (enabled: true, default: false)
- 언어 추가 시 설문의 모든 텍스트 필드에 해당 언어 키 자동 추가
- 언어 비활성화 시 enabled=false로 설정 (번역 데이터는 보존)

#### 4.7 다국어 비활성화 시 확인

- 다국어 비활성화 시 확인 모달 표시
- 확인 시 모든 번역 데이터 삭제 (languages 배열 초기화)
- 파괴적 작업이므로 destructive 스타일 적용

#### 4.8 Language Switch (언어 전환 UI)

- showLanguageSwitch=true: 응답자가 설문 진행 중 언어를 전환할 수 있는 UI 표시
- 활성화된 언어(enabled)가 2개 이상일 때만 토글 가능
- 활성 언어 1개 이하이면 토글 비활성화

#### 4.9 Link Survey에서의 언어 지정 (?lang=)

- URL 파라미터: ?lang=ko, ?lang=ar-SA 등
- 유효한 언어 코드가 전달되면 해당 언어로 설문 표시
- 무효한 코드이면 기본 언어(default)로 fallback

#### 4.10 RTL (Right-to-Left) 언어 지원

**지원 RTL 언어 목록**:

| 코드 | 언어 |
|------|------|
| ar | 아랍어 (일반) |
| ar-SA | 아랍어 (사우디아라비아) |
| ar-EG | 아랍어 (이집트) |
| ar-AE | 아랍어 (UAE) |
| ar-MA | 아랍어 (모로코) |
| he | 히브리어 |
| fa | 페르시아어 |
| ur | 우르두어 |

**RTL 판별 로직**:
1. 다국어 설정이 없는 경우: Welcome Card 헤드라인 텍스트의 유니코드 방향성으로 판별
2. 다국어 설정이 있는 경우: 언어 코드의 기본 코드(base code)가 RTL 언어 목록에 있는지 확인

**RTL 적용 범위**:
- 로고 위치 방향
- 설문 컴포넌트 전체에 dir="rtl" 속성 적용

#### 4.11 Enterprise 라이선스 검증

- Formbricks Cloud: 유료 플랜에서 사용 가능
- Self-hosted: Enterprise 라이선스 키의 다국어 설문 기능 활성화 필요

#### 4.12 번역 필드 관리

1. 새 언어 추가 시 설문의 모든 다국어 문자열 필드에 해당 언어 키 자동 추가
2. 기존 텍스트는 빈 문자열로 초기화
3. 번역은 설문 에디터에서 언어별로 입력

#### 4.13 텍스트 필드 언어별 검증

- 활성화된 모든 언어에 대해 텍스트가 비어있지 않은지 검증
- HTML 콘텐츠인 경우 실제 텍스트 추출 후 확인

## 6. 정책/제약

| 항목 | 제약 값 |
|------|--------|
| 다국어 문자열 필수 키 | "default" |
| RTL 지원 언어 | ar, ar-SA, ar-EG, ar-AE, ar-MA, he, fa, ur (8개) |
| 다국어 활성화 최소 언어 수 | 프로젝트에 2개 이상 |
| Language Switch 최소 활성 언어 | 2개 이상 |
| 언어 코드 URL 파라미터 | ?lang= |
| 라이선스 기능 플래그 | 다국어 설문 (multiLanguageSurveys) |
| 언어 ID 형식 | CUID2 |
| 프로젝트-언어 관계 | 프로젝트 레벨에서 관리 |
| 다국어 비활성화 시 | 모든 번역 삭제 (확인 필요) |
| 기본 언어 키 | "default" (언어 코드가 아닌 고정 문자열) |
