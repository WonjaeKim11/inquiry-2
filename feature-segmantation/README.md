## 5. 문서 템플릿 구조

각 문서는 다음 7개 섹션으로 구성됩니다:

1. **목적/배경** — 기능의 존재 이유와 비즈니스 가치
2. **범위 (In-scope / Out-of-scope)** — 명확한 경계 정의
3. **사용자/이해관계자** — 기능을 사용하는 역할
4. **기능 요구사항** — 구현 상세 포함 (타입, 스키마, API, 플로우)
5. **비기능 요구사항** — 성능, 보안, 가용성 관련
6. **정책/제약** — 코드에서 추출한 실제 값 (한도, 타임아웃, 유효성 검사 규칙 등)
7. **수용 기준 (Acceptance Criteria)** — 검증 가능한 통과 조건
