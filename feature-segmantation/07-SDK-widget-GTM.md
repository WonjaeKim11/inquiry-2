# SDK, 위젯 & GTM 통합 -- 요구사항 명세서

> **문서번호**: FSD-007 | **FR 범위**: FR-010, FR-042
> **라이선스**: Community

---

## 1. 목적/배경

Formbricks JS SDK는 웹 애플리케이션에 설문을 삽입하기 위한 클라이언트 사이드 런타임이다. Environment ID 기반으로 초기화되며, Action 기반 트리거링, 위젯 배치 제어, 오버레이 스타일링, per-survey 오버라이드를 지원한다. GTM(Google Tag Manager)과의 통합을 통해 태그 매니저 기반 배포도 가능하다.

## 2. 범위

### In-scope
- JS SDK 초기화 및 설정 흐름
- 위젯 배치(Placement) 5가지 유형
- 오버레이(Overlay) 3가지 유형
- 설문 표시 모드 (modal, inline)
- 외부 클릭 닫기 옵션
- 설문별 프로젝트 설정 오버라이드
- No-code Action 감지 메커니즘
- GTM 통합 방식
- 상태 관리 (Environment State, User State)
- 에러 상태 관리

### Out-of-scope
- 서버 사이드 API 엔드포인트 구현 상세
- 설문 응답 수집/저장 로직
- Contact/Segment 서버 로직

## 3. 사용자/이해관계자

| 역할 | 설명 |
|------|------|
| 프론트엔드 개발자 | SDK를 자사 웹앱에 통합 |
| 마케팅 담당자 | GTM을 통한 설문 배포 관리 |
| SDK 런타임 | 자동 환경 동기화 및 설문 렌더링 |

## 4. 기능 요구사항

### FR-010: SDK 초기화 및 구성

#### 4.1 초기화 입력

SDK 초기화에 필요한 입력 데이터:

| 필드 | 타입 | 필수 여부 | 설명 |
|------|------|-----------|------|
| Environment ID | 문자열 | 필수 | Formbricks Environment의 CUID |
| App URL | 문자열 | 필수 | Formbricks 서버 URL |

고급 초기화 (식별된 사용자)의 추가 필드:
- 사용자 ID (문자열)
- 속성 (키-값 쌍)

#### 4.2 SDK 초기화 흐름

SDK 초기화 과정(설정 처리):

1. **디버그 모드 확인**: 디버그 여부를 파악하고, 디버그 시 로그 레벨을 debug로 설정
2. **LocalStorage 마이그레이션**: Legacy 포맷(환경 상태/개인 상태)에서 새 포맷(환경/사용자)으로 자동 마이그레이션
3. **이미 초기화 여부 확인**: 초기화 완료 상태면 스킵
4. **에러 상태 확인**: 에러 상태이면 만료 시간 확인 후 스킵 또는 재시도
5. **필수 필드 검증**: Environment ID, App URL 누락 시 필수 필드 누락 에러 반환
6. **기존 설정 확인**: 유효한 설정이 존재하면 만료 여부 확인 후 증분 동기화
7. **환경 상태 가져오기**: 서버에서 환경 상태 조회
8. **사용자 상태 처리**: 사용자 ID 존재 시 서버에 업데이트 전송
9. **설문 필터링**: 환경 상태와 사용자 상태 기반으로 설문 필터링 실행
10. **이벤트 리스너 등록**: 이벤트 리스너 및 정리 이벤트 리스너 등록
11. **초기화 완료**: 초기화 완료 플래그 설정

#### 4.3 SDK 설정 상태

SDK 내부 설정 상태 구조:

| 필드 | 타입 | 설명 |
|------|------|------|
| Environment ID | 문자열 | 초기화에 사용된 환경 ID |
| App URL | 문자열 | Formbricks 서버 URL |
| 환경 상태 | Environment State 구조 | 환경 관련 데이터 |
| 사용자 상태 | User State 구조 | 사용자 관련 데이터 |
| 필터링된 설문 목록 | 설문 배열 | 조건에 맞는 설문 목록 |
| 상태 | success 또는 error + 만료 시각 | SDK 상태 정보 |

#### 4.4 Environment State

환경 상태에 포함되는 데이터:

| 필드 | 설명 |
|------|------|
| 만료 시각 | 상태 캐시 만료 시점 |
| 설문 목록 | 현재 환경의 설문 데이터 |
| Action Class 목록 | 현재 환경의 액션 정의 |
| 프로젝트 정보 | ID, 재접촉 일수, 외부 클릭 닫기, 오버레이, 위젯 배치, In-app 설문 브랜딩, 스타일링 |
| reCAPTCHA 사이트 키 | 스팸 방지용 (선택) |

#### 4.5 User State

사용자 상태에 포함되는 데이터:

| 필드 | 설명 |
|------|------|
| 만료 시각 | 상태 캐시 만료 시점 (null 허용) |
| 사용자 ID | 사용자 식별자 (null 허용) |
| 연락처 ID | 연락처 식별자 (null 허용) |
| 세그먼트 목록 | 소속 세그먼트 ID 배열 |
| 표시 이력 | 설문 ID와 표시 일시의 배열 |
| 응답 목록 | 응답한 설문 ID 배열 |
| 마지막 표시 시각 | 마지막으로 설문이 표시된 시각 (null 허용) |
| 언어 | 사용자 언어 (선택) |

#### 4.6 설정 영속화

- Singleton 패턴으로 설정 인스턴스 관리
- LocalStorage에 설정 저장/로드
- LocalStorage 키: "formbricks-js"

### FR-010: 위젯 배치 및 표시

#### 4.7 Placement 5가지 유형

| 배치 위치 | 설명 |
|-----------|------|
| bottomLeft | 화면 좌하단에 위젯 표시 |
| bottomRight | 화면 우하단에 위젯 표시 (기본값) |
| topLeft | 화면 좌상단에 위젯 표시 |
| topRight | 화면 우상단에 위젯 표시 |
| center | 화면 중앙에 위젯 표시 |

#### 4.8 Overlay 3가지 유형

| 오버레이 | 설명 |
|----------|------|
| none | 배경 오버레이 없음 (기본값) |
| light | 밝은 반투명 배경 오버레이 |
| dark | 어두운 반투명 배경 오버레이 |

#### 4.9 표시 모드

SDK 위젯의 표시 모드:
- **modal**: 오버레이와 함께 팝업 형태로 표시 (app 타입 설문의 기본)
- **inline/fullwidth**: 페이지에 직접 삽입 (link 타입 설문의 기본)

app 타입 설문은 modal 프리뷰, link 타입 설문은 fullwidth 프리뷰로 결정됨.

#### 4.10 외부 클릭 닫기 옵션

- Project 레벨에서 설정 (기본값: true)
- 설문 외부 영역 클릭 시 설문을 닫을지 여부를 제어
- 설문별 프로젝트 설정 오버라이드에서 per-survey 오버라이드 가능
- 우선순위: 설문별 오버라이드 > 프로젝트 기본 설정

#### 4.11 설문별 프로젝트 설정 오버라이드 (Per-Survey Override)

개별 설문에서 프로젝트 기본 설정을 오버라이드할 수 있는 속성:

| 오버라이드 항목 | 타입 | 설명 |
|----------------|------|------|
| 브랜드 색상 | HEX Color | 브랜드 색상 |
| 하이라이트 테두리 색상 | HEX Color | 하이라이트 테두리 색상 |
| 위젯 배치 | Placement enum | 위젯 배치 위치 |
| 외부 클릭 닫기 | boolean | 외부 클릭 닫기 |
| 오버레이 | Overlay enum | 오버레이 유형 |

우선순위 적용 규칙:
- 설문별 오버라이드 값이 있으면 우선 적용
- 없으면 프로젝트 기본값 사용

#### 4.12 위젯 렌더링

위젯 렌더링 처리 과정:

1. **중복 실행 방지**: 설문 실행 중 플래그로 동시 설문 실행 차단
2. **표시 확률 확인**: 설정된 경우 확률 기반 표시 여부 결정
3. **Hidden Fields 처리**: Hidden Fields 값 처리
4. **다국어 처리**: 설문 언어 수가 2개 이상이면 사용자 언어와 매칭
5. **딜레이 적용**: 설문 딜레이(초) 후 렌더링
6. **설문 패키지 로드**: 설문 렌더링 함수 호출

설문 패키지 로드 방식:
- 이미 로드된 설문 패키지가 있으면 재사용
- 없으면 설문 스크립트를 동적으로 삽입하여 로드

위젯 컨테이너: 전용 컨테이너 div 요소를 body에 추가/제거

#### 4.13 설문 트리거 및 표시 흐름

설문 트리거 처리 흐름:
1. 표시 확률 확인 -- 확률 미달 시 스킵
2. Hidden Fields 처리
3. 위젯 렌더링 호출

설문 렌더링 시 전달되는 주요 파라미터:
- App URL, Environment ID, 연락처 ID, 액션, 설문 데이터
- 브랜딩 활성화 여부, 외부 클릭 닫기, 오버레이
- 언어 코드, 위젯 배치, 스타일링
- Hidden Fields 데이터, reCAPTCHA 설정
- 표시 생성/응답 생성/닫기 콜백

#### 4.14 No-Code Action 감지

4가지 No-Code Action 이벤트 리스너:

**1. Page URL 이벤트**:
- 감지 이벤트: hashchange, popstate, pushstate, replacestate, load
- SPA 라우팅 변경 감지를 위해 history.pushState를 래핑
- URL 필터와 커넥터(or/and)로 매칭 여부 판단

**2. Click 이벤트**:
- 문서 레벨 click 이벤트 등록
- 대상 요소의 CSS 선택자 또는 내부 HTML 매칭으로 트리거 판단

**3. Exit Intent 이벤트**:
- body 레벨 mouseleave 이벤트 등록
- 마우스 Y좌표가 0 이하 (뷰포트 상단 벗어남) 시 트리거
- URL 필터 추가 검증

**4. 50% Scroll 이벤트**:
- window 레벨 scroll 이벤트 등록
- 스크롤 위치 / (전체 높이 - 화면 높이) >= 0.5 조건
- 한 번 트리거 후 스크롤 위치가 0으로 돌아가야 재트리거 가능

### FR-042: GTM 통합

#### 4.15 GTM 통합 방식

GTM(Google Tag Manager) 통합은 Custom HTML 태그를 통해 수행:

1. GTM 컨테이너에서 Custom HTML 태그 생성
2. Formbricks SDK 스크립트를 삽입하고 Environment ID와 App URL로 초기화
3. Environment ID 기반 초기화로 해당 환경의 설문만 로드

#### 4.16 에러 상태 관리

- 에러 발생 시 SDK를 에러 상태로 전환
- 에러 상태 지속 시간: **10분** (600,000ms)
- 에러 상태 동안 SDK 재초기화가 차단됨
- 디버그 모드 활성화 시 에러 상태 무시하고 재초기화 가능
- 에러 상태 진입 시: 설정 업데이트 및 정리(tearDown) 처리 수행

#### 4.17 정리 처리 (tearDown/로그아웃)

정리 처리 시 수행되는 작업:
- User State를 기본값으로 리셋
- 필터링된 설문 목록 재계산
- 활성 설문 닫기

#### 4.18 설문 닫기

설문 닫기 시 수행되는 작업:
- DOM에서 위젯 컨테이너 제거
- 설문 필터 재계산
- 설문 실행 중 플래그 리셋

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 번들 크기 | 설문 패키지는 UMD + ESM 빌드, 외부 스크립트로 동적 로드 |
| 캐싱 | LocalStorage에 설정 캐시, 만료 시각 기반 만료 관리 |
| 에러 복구 | 에러 상태 10분 지속 후 자동 재시도, 디버그 모드 시 즉시 재시도 |
| SPA 지원 | history.pushState 래핑으로 SPA 라우팅 변경 감지 |
| 동시 실행 방지 | 설문 실행 중 플래그로 한 번에 하나의 설문만 표시 |
| CSP 호환 | nonce 지원 |

## 7. 수용 기준 (Acceptance Criteria)

1. **AC-010-01**: SDK는 Environment ID와 App URL 없이 초기화할 수 없다 (필수 필드 누락 에러).
2. **AC-010-02**: 초기화 후 LocalStorage에 formbricks-js 키로 설정이 저장된다.
3. **AC-010-03**: 위젯은 5가지 Placement 위치 중 하나에 정확히 렌더링된다.
4. **AC-010-04**: Overlay 유형에 따라 배경이 none/light/dark로 표시된다.
5. **AC-010-05**: 외부 클릭 닫기가 true이면 위젯 외부 클릭 시 설문이 닫힌다.
6. **AC-010-06**: Survey의 설문별 오버라이드가 Project 기본 설정보다 우선 적용된다.
7. **AC-010-07**: No-code pageView 액션은 URL 필터 규칙에 따라 정확히 트리거된다.
8. **AC-010-08**: No-code click 액션은 CSS 선택자 또는 내부 HTML 매칭으로 트리거된다.
9. **AC-010-09**: No-code exitIntent 액션은 마우스가 뷰포트 상단을 벗어날 때 트리거된다.
10. **AC-010-10**: No-code fiftyPercentScroll 액션은 50% 스크롤 도달 시 한 번만 트리거된다.
11. **AC-010-11**: 이미 설문이 표시 중이면 새 설문이 트리거되지 않는다.
12. **AC-010-12**: 에러 상태 진입 후 10분 내에는 SDK 재초기화가 스킵된다.
13. **AC-010-13**: 표시 확률이 설정된 설문은 확률에 따라 표시 여부가 결정된다.
14. **AC-042-01**: GTM Custom HTML 태그를 통해 SDK를 초기화하고 설문을 트리거할 수 있다.
15. **AC-042-02**: GTM 통합은 Environment ID 기반으로 해당 환경의 설문만 로드한다.
