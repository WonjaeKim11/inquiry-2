# 응답 알림(Notification) — 요구사항 명세서

> **문서번호**: FSD-028 | **FR 범위**: FR-046
> **라이선스**: Community (기본 기능)

---

## 1. 목적/배경

Formbricks의 응답 알림 기능은 설문 응답이 완료될 때 관련 사용자에게 이메일로 알림을 보내는 기능이다. 사용자는 설문별로 알림을 활성화/비활성화할 수 있으며, Organization 단위로 새 설문 자동 구독 여부를 설정할 수 있다. 현재는 이메일 알림만 지원하며, 인앱 알림은 제공하지 않는다.

---

## 2. 범위 (In-scope / Out-of-scope)

### In-scope
- 설문 응답 완료 시 이메일 알림 전송
- 설문별 알림 토글 (on/off)
- Organization 단위 자동 구독 설정
- 설문 생성 시 생성자 자동 구독
- 이메일 내 구독 해제 링크
- Production 환경 설문만 알림 대상

### Out-of-scope
- 인앱(in-app) 알림
- 응답 생성(responseCreated) 트리거 알림
- Slack/Discord 등 외부 채널 알림 (Webhook으로 대체)
- 알림 빈도 제한 (매 응답마다 발송)
- 알림 내용 커스터마이징

---

## 3. 사용자/이해관계자

| 역할 | 설명 |
|------|------|
| Organization Owner/Manager | 모든 프로젝트의 설문 알림 관리 가능 |
| Team Member | 팀을 통해 접근 가능한 프로젝트의 설문 알림 관리 |
| Survey Creator | 설문 생성 시 자동 구독 (Organization 정책에 따라) |
| Billing Role | 알림 설정 접근 불가 (notifications 페이지 제외) |

## 4. 기능 요구사항

### FR-046-01: 알림 설정 데이터 모델

**사용자 알림 설정 구조**:

사용자의 알림 설정은 User 모델 내 JSON 필드에 저장되며, 다음 두 가지 요소로 구성된다:

- **alert**: 설문 ID를 키로, 알림 활성화 여부를 값으로 가지는 레코드. 예: { "survey_id_1": true, "survey_id_2": false }
- **자동 구독 해제 Organization 목록**: 자동 구독을 해제한 Organization의 ID 배열 (선택)

### FR-046-02: 알림 설정 UI

#### 알림 설정 페이지 구조

1. **Organization 단위 자동 구독 토글**: Organization별로 "Auto-subscribe to new surveys" 설정
2. **설문별 알림 토글**: 각 설문에 대해 "Every Response" 알림 on/off

#### Production 환경 필터

알림 설정 페이지에서는 **production 환경의 설문만** 표시된다.

#### 접근 권한

알림 설정 페이지 접근은 billing 역할을 제외한 모든 멤버에게 허용된다:
- owner/manager: 모든 프로젝트 접근
- 기타 멤버: 팀 기반 접근 (팀을 통해 배정된 프로젝트만)

#### 설정 초기화

페이지 로드 시 현재 사용자의 모든 설문에 대한 알림 상태를 초기화한다. 레거시 형식과의 호환성도 유지하며, 기존에 설정이 없는 설문은 비활성(false)으로 초기화된다.

### FR-046-03: 알림 스위치 컴포넌트

두 가지 알림 유형을 토글하는 스위치 컴포넌트:

#### 알림 유형

| 알림 유형 | 설명 | 동작 |
|--------------------|------|------|
| 설문별 응답 알림 | 설문별 응답 알림 | 해당 설문의 알림 활성화/비활성화 토글 |
| Organization 자동 구독 | Organization 자동 구독 | 자동 구독 해제 Organization 목록에서 추가/제거 |

#### 자동 해제 기능

URL 파라미터를 통한 자동 알림 해제를 지원한다. 이메일 내 "Unsubscribe" 링크에서 사용되며, 해당 URL로 접속하면 자동으로 알림을 비활성화한다.

### FR-046-04: 설문 생성 시 자동 구독

설문 생성 시 다음 조건에 따라 자동 구독이 처리된다:

**자동 구독 조건**:
1. 설문 생성자만 자동 구독 (Organization 전체 멤버가 아님)
2. 해당 Organization에 대해 자동 구독 해제 목록에 포함되지 않은 경우만

자동 구독이 적용되면 새 설문에 대해 알림이 자동으로 활성화된다.

### FR-046-05: 알림 이메일 발송

응답 완료(responseFinished) 이벤트 발생 시 알림 이메일이 발송된다.

#### 수신자 조건

1. 해당 환경의 Organization 멤버
2. owner 또는 manager 역할이거나, 해당 프로젝트에 팀을 통해 접근 가능한 사용자
3. 해당 설문에 대한 알림이 활성화(true)된 사용자

#### 이메일 발송

- 이메일은 병렬 발송되며, 개별 실패가 전체에 영향 주지 않음
- 사용자의 locale 설정에 따라 이메일 언어 결정
- 현재까지의 응답 수 정보가 이메일에 포함됨

### FR-046-06: 알림 트리거 조건

| 이벤트 | 알림 발송 여부 | 비고 |
|--------|--------------|------|
| responseFinished (응답 완료) | O | 설문 응답 완료 시 |
| responseCreated (응답 생성) | X | 응답 생성 시점에는 알림 없음 |

Pipeline의 알림 로직은 응답 완료(responseFinished) 이벤트에서만 실행된다.

### FR-046-07: 구독 해제

#### 이메일 내 구독 해제

이메일에 포함된 구독 해제 링크를 클릭하면 알림 설정 페이지로 이동하며, 해당 설문의 알림이 자동으로 비활성화된다.

#### Organization 자동 구독 해제

자동 구독 해제 Organization 목록에 Organization ID를 추가하면, 해당 Organization의 새 설문 생성 시 자동 구독되지 않는다.

## 5. 비기능 요구사항

| ID | 항목 | 내용 |
|----|------|------|
| NFR-N01 | 신뢰성 | 병렬 발송으로 개별 이메일 발송 실패가 다른 알림에 영향 주지 않음 |

## 6. 정책/제약

| 항목 | 값 |
|------|-----|
| 알림 채널 | 이메일만 |
| 알림 트리거 | 응답 완료(responseFinished)만 |
| 자동 구독 대상 | 설문 생성자만 |
| 알림 대상 환경 | Production만 |
| 제외 역할 | billing |
| 수신자 권한 조건 | owner/manager 또는 팀 멤버 |
| 알림 설정 저장 | User 모델의 JSON 필드 |
| 설정 구조 | alert(설문별 boolean 레코드) + 자동 구독 해제 Organization 목록 |
| Pipeline 인증 | 내부 인증 키 (CRON_SECRET) |
| 이메일 발송 방식 | 병렬 (개별 실패 격리) |

## 7. 수용 기준 (Acceptance Criteria)

| AC-ID | 기준 |
|-------|------|
| AC-028-01 | 설문 응답 완료 시 해당 설문에 알림이 활성화된 모든 적격 사용자에게 이메일이 발송된다 |
| AC-028-02 | 알림이 비활성화된 사용자에게는 이메일이 발송되지 않는다 |
| AC-028-03 | 설문별 알림 토글 변경 시 알림 설정이 즉시 업데이트된다 |
| AC-028-04 | Organization 자동 구독 해제 시 새 설문 생성 시 자동 구독되지 않는다 |
| AC-028-05 | Organization 자동 구독이 활성화된 상태에서 새 설문 생성 시 생성자가 자동 구독된다 |
| AC-028-06 | billing 역할의 사용자는 알림 설정 페이지에서 설문을 볼 수 없다 |
| AC-028-07 | development 환경의 설문은 알림 설정 페이지에 표시되지 않는다 |
| AC-028-08 | 이메일 내 구독 해제 링크 클릭 시 해당 설문의 알림이 비활성화된다 |
| AC-028-09 | 이메일은 수신자의 locale 설정에 맞는 언어로 발송된다 |
| AC-028-10 | 개별 이메일 발송 실패가 다른 수신자의 알림 발송에 영향을 주지 않는다 |
| AC-028-11 | responseCreated 이벤트에서는 알림 이메일이 발송되지 않는다 |
| AC-028-12 | 레거시 알림 설정 형식도 올바르게 처리된다 |
